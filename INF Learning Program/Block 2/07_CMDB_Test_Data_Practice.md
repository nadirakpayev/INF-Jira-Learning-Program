# Практическое задание 7. Тестовые данные для CMDB

## Ситуация

Перед тем как проектировать и разворачивать CMDB, участнику нужен единый набор тестовых данных. Эти данные должны быть похожи на реальные, но оставаться учебными и безопасными. Они будут использоваться и в задании по моделированию CMDB, и в задании по PostgreSQL и pgAdmin.

## Цель

Подготовить и освоить единый учебный датасет для CMDB, который затем будет использоваться в следующих заданиях блока 3.

## Роль в программе

Это отдельный блок, который готовит данные для всех CMDB-упражнений. Без него задания по CMDB и локальной БД выполняются на произвольных данных и хуже проверяются.

## Что нужно сделать

1. Откройте файл [CMDB_Test_Data_Reference.md](CMDB_Test_Data_Reference.md).
2. Изучите структуру учебного датасета:
   - сервисы;
   - системы;
   - модули;
   - конфигурационные единицы;
   - сотрудники;
   - рабочие группы;
   - технологические окна.
3. Определите, какие сущности из файла должны стать:
   - отдельными сущностями в CMDB-модели;
   - таблицами в PostgreSQL;
   - справочниками или `Components` в Jira.
4. Подготовьте карту использования данных:
   - какие записи вы будете использовать в задании по моделированию;
   - какие записи загрузите в локальную БД;
   - какие записи затем будете использовать в сценариях major incident, problem и change.
5. Проверьте согласованность датасета:
   - у каждого модуля должна быть система;
   - у системы должен быть владелец;
   - у системы должна быть рабочая группа;
   - у технологического окна должна быть привязка к системе.
6. Подготовьте краткую заметку, как именно вы будете использовать этот датасет в заданиях `08` и `09`.

## Cloud-акцент

- Сам датасет не настраивается в Jira напрямую.
- Его задача: дать устойчивую основу для моделирования данных и для будущего отражения части сущностей в Jira Cloud.

## Артефакты

- Изученный и принятый к работе учебный датасет.
- Карта использования данных в следующих заданиях.
- Проверка согласованности данных.

## Критерии приемки

- Участник понимает структуру и назначение учебного датасета.
- Участник может объяснить, как эти данные будут использоваться в моделировании CMDB и в PostgreSQL.
- В следующих заданиях используется именно этот датасет, а не случайные данные.
