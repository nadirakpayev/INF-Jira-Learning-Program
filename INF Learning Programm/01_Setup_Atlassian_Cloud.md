# Практическое задание 1. Создание учебной среды в Atlassian Cloud

## Ситуация

Вам нужно развернуть собственную учебную площадку, на которой позже будут настраиваться процессы поддержки, изменений, проблем, доступности и справочники. Площадка должна быть подготовлена так, чтобы на ней можно было безопасно моделировать реальный ITSM-ландшафт без привязки к корпоративной инфраструктуре.

## Цель

Подготовить личную бесплатную среду Jira Service Management Cloud для прохождения всей программы без использования trial и Assets.

## Что нужно сделать

1. Создайте личный Atlassian account.
2. Создайте новый бесплатный Jira Service Management Cloud site.
3. Во время первичной настройки создайте первый service project и выберите company-managed вариант.
4. Задайте нейтральное имя сайту, проекту и help center без использования названий из исходных документов.
5. Создайте минимум 5 тестовых ролей:
   - администратор платформы;
   - агент Service Desk;
   - менеджер процесса;
   - согласующий;
   - заявитель.
6. Подготовьте тестовые учетные записи или зафиксируйте матрицу ролей, если отдельные пользователи пока не создаются.
7. Подготовьте соглашение о том, как в учебной среде будут использоваться стандартные поля Jira:
   - Components для систем, сервисов или технических доменов;
   - select list для модулей, КЕ, типов простоя, видов решения и других справочников.
8. Проверьте, что у вас есть доступ к следующим разделам:
   - проекту обслуживания;
   - portal/help center;
   - очередям;
   - automation;
   - SLA;
   - forms;
   - components;
   - custom fields.
9. Создайте отдельную страницу заметок или файл, где будете фиксировать:
   - URL сайта;
   - URL портала;
   - имя проекта;
   - список ролей;
   - какие справочники моделируются через Components;
   - какие справочники моделируются через select list.

## Cloud-акцент

- Работайте только в Jira Service Management Cloud.
- Для учебных задач используйте company-managed service project, так как он лучше подходит для сложных workflow, SLA, approvals и общей администрируемости.
- Базовый сценарий программы начинается и заканчивается на бесплатном Cloud-сайте.
- Компоненты проекта используются как основной механизм технической классификации там, где это уместно.
- Для остальных справочников используйте select list и согласованные правила именования.

## Артефакты

- Подготовленный cloud-site.
- Один service project для ITSM-упражнений.
- Матрица ролей.
- Краткий журнал ограничений среды.

## Критерии приемки

- Atlassian account создан и подтвержден.
- Site открывается и доступен владельцу.
- В проекте можно создать заявку через портал и через agent view.
- Участник понимает, какие данные будут храниться в Components, а какие в select list.

## Дополнительное задание

Подготовьте короткую памятку для нового участника команды: как зайти в сайт, где находится портал, где смотреть automation и где настраиваются Components и select list.
