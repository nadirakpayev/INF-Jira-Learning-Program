# Improved Screening and Onboarding Package

Этот пакет описывает не отдельное разовое тестовое задание, а целевую модель программы входа в работу для `Jira Service Management Cloud`.

Ключевой принцип:
- `Блок 1` и `Блок 2` используются как screening-этап и дают решение, подходит ли кандидат;
- `Блок 3` и `Блок 4` выполняются после положительного решения и используются как onboarding-подготовка;
- `Source Docs` не входят в оцениваемый контур и не являются обязательным чтением для кандидата.

## Как использовать пакет

1. Кандидату выдаются только материалы из `Improved`.
2. Кандидат сдает `Блок 1`, затем `Блок 2`.
3. После проверки `Блока 2` принимается решение `PASS / FAIL`.
4. Только при `PASS` кандидат продолжает на `Блок 3` и `Блок 4`.

## Что считается screening

- `Блок 1`: базовая работа в `JSM Cloud`:
  - собственное облако;
  - сервисный каталог;
  - `Incident`;
  - `Service Request`;
  - `SLA` и `Queues`.
- `Блок 2`: базовая работа с данными и расширенной логикой:
  - тестовые данные `CMDB`;
  - моделирование данных;
  - локальный `PostgreSQL + pgAdmin`;
  - связка `Major Incident -> Problem -> Change`;
  - `SLA` и `Queues` для второго блока.

## Что считается onboarding

- `Блок 3`: `Availability Management` в `Jira SM Cloud`.
- `Блок 4`: проектирование интеграционных моделей и итоговая консолидация.

## Роль Source Docs

`Source Docs/` используются только как внутренний источник контекста:
- как примеры того, как выглядят реальные регламенты и ТЗ;
- как источник терминов и сценариев;
- как референс для лидов и методистов.

Кандидат не должен:
- читать весь архив `Source Docs`;
- извлекать оттуда требования;
- использовать его как основной источник истины по заданию.

## Состав пакета

- `assignment.md` — staged-постановка задания по блокам;
- `submission_format.md` — формат сдачи по блокам;
- `evaluation_criteria.md` — правила `PASS / FAIL` и правила завершения onboarding;
- `scoring_rubric.md` — 100-балльная шкала для screening;
- `review_checklist.md` — чеклист ревьюера по блокам;
- `risk_points.md` — точки риска и возможные застревания кандидата;
- `faq_for_candidates.md` — ответы на типовые вопросы;
- `anti_cheating_notes.md` — меры против поверхностного прохождения;
- `improvement_notes.md` — кратко, что изменено в логике программы;
- `analysis_report.md` — аудит текущей программы и объяснение, почему пакет перестроен именно так.
