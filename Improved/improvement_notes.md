# Что изменено относительно текущей программы

## 1. Изменена сама логика пакета

Было:
- учебная программа воспринималась как единый длинный трек;
- screening и onboarding были смешаны.

Стало:
- программа разделена на screening и onboarding;
- `Блок 1` и `Блок 2` дают решение по кандидату;
- `Блок 3` и `Блок 4` выполняются только после `PASS`.

## 2. Assessment перестал быть “одним большим заданием”

Было:
- `Improved`-пакет описывал один bounded assessment.

Стало:
- `Improved`-пакет описывает staged-прохождение по блокам;
- проверка идет по этапам, а не одной финальной кучей.

## 3. Source Docs выведены из оценочного контура

Было:
- архив мог восприниматься как часть входного задания.

Стало:
- `Source Docs` объявлены только внутренним контекстом и примерами для методистов и лидов;
- кандидат работает только по материалам из `Improved`.

## 4. Screening завязан на Jira SM Cloud

Было:
- в контуре ощущалась логика “переноса DC-задачи”.

Стало:
- пакет прямо построен вокруг `Jira Service Management Cloud`;
- ограничения `Cloud` считаются частью задания, а не исключением.

## 5. Усилена воспроизводимость проверки

Добавлено:
- staged submission;
- отдельные критерии сдачи `Блока 1` и `Блока 2`;
- рубрика на `100` баллов только для screening;
- completion-статусы для onboarding-блоков.
