# Базовый словарь терминов

У каждого термина есть короткое определение и ссылка, куда можно углубиться.

## 1. Jira Service Management Cloud

Платформа Atlassian для управления сервисными запросами, инцидентами и смежными ITSM-процессами в облачной модели.

Углубиться:
- [What is Jira Service Management?](https://support.atlassian.com/jira-service-management-cloud/docs/what-is-jira-service-management/)

## 2. Company-managed project

Тип проекта Jira, где настройки управляются централизованно и лучше подходят для более сложных схем, ролей, workflow и SLA.

Углубиться:
- [Company-managed vs team-managed](https://support.atlassian.com/jira-software-cloud/docs/what-are-team-managed-and-company-managed-projects/)

## 3. Portal

Клиентская витрина сервиса, через которую пользователь подает запросы и отслеживает их статус.

Углубиться:
- [What is a portal?](https://support.atlassian.com/jira-service-management-cloud/docs/about-the-portal-and-help-center/)

## 4. Help center

Общая точка входа для пользователей, где могут быть собраны порталы, статьи и другие ресурсы помощи.

Углубиться:
- [What is a portal?](https://support.atlassian.com/jira-service-management-cloud/docs/about-the-portal-and-help-center/)

## 5. Request channel

Канал, через который запрос попадает в систему: портал, email, chat, widget и другие варианты.

Углубиться:
- [What are request channels?](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-request-channels/)

## 6. Work item

Внутреннее представление единицы работы для агентов и администраторов Jira Service Management.

Углубиться:
- [About requests and work items](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-issues-and-requests/)

## 7. Request

То, как единица работы представляется конечному пользователю на портале или в help center.

Углубиться:
- [About requests and work items](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-issues-and-requests/)

## 8. Request type

Пользовательский тип обращения, который определяет, как запрос выглядит на портале и какие данные собираются при создании.

Углубиться:
- [What are request types?](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-request-types-in-a-service-project/)
- [Request types in ITSM](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-request-types-in-itsm/)

## 9. Work type

Внутренний тип работы, который задает базовую структуру запроса: поля, workflow и другие фундаментальные настройки.

Углубиться:
- [Difference between request types and work types](https://support.atlassian.com/jira-service-management-cloud/docs/whats-the-difference-between-request-types-and-issue-types/)

## 10. Service request

Запрос пользователя на что-то новое: доступ, оборудование, информацию или стандартную услугу.

Углубиться:
- [What are service requests?](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-service-requests/)
- [What is a service request?](https://www.atlassian.com/itsm/service-request-management)

## 11. Incident

Незапланированное событие, которое нарушает сервис или снижает качество его работы.

Углубиться:
- [Incidents](https://support.atlassian.com/jira-service-management-cloud/docs/incidents/)

## 12. Queue

Фильтрованный рабочий список заявок, который помогает агентам видеть нужный срез работы.

Углубиться:
- [What are queues?](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-queues/)
- [About queues](https://support.atlassian.com/jira-service-management-cloud/docs/check-out-your-queues/)

## 13. SLA

Измеряемое целевое обязательство по сервису, например по времени первой реакции или времени решения.

Углубиться:
- [About SLAs](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-slas-and-how-do-i-view-them-in-my-service-project/)
- [SLA overview](https://www.atlassian.com/itsm/service-request-management/slas)

## 14. Approval

Шаг процесса, на котором запрос должен быть одобрен до перехода дальше по workflow.

Углубиться:
- [What are approvals?](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-approvals/)

## 15. Automation rule

Правило автоматизации, состоящее из триггера, условий и действий.

Углубиться:
- [Create automation rules in Jira](https://support.atlassian.com/cloud-automation/docs/create-and-edit-jira-automation-rules/)

## 16. Component

Проектный классификатор Jira, который помогает группировать работу по системам, доменам или областям ответственности.

Углубиться:
- [What are Jira components?](https://support.atlassian.com/jira-software-cloud/docs/organize-work-with-components/)

## 17. Form

Гибкая форма в Jira Service Management для сбора структурированных данных, в том числе с условной логикой.

Углубиться:
- [What are forms?](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-forms/)
- [Use forms in Jira Service Management](https://support.atlassian.com/jira-service-management-cloud/docs/use-forms-in-jira-service-management/)

## 18. Jira field

Поле Jira, которое хранится в системе и участвует в workflow, automation, JQL и отчетности.

Углубиться:
- [Form fields and Jira fields](https://support.atlassian.com/jira-service-management-cloud/docs/what-are-form-fields-and-jira-fields/)

## 19. Select list

Поле со списком предопределенных значений, удобное для небольших справочников и классификаторов.

Углубиться:
- [Customize fields in your IT service project](https://support.atlassian.com/jira-service-management-cloud/docs/customize-fields-in-your-it-service-project/)

## 20. CMDB

Упорядоченная модель данных о сервисах, системах, конфигурационных единицах и их связях.

Углубиться:
- [ITSM Glossary](https://success.atlassian.com/solution-paths/solution-guides/itsm-solution-delivery-guide/itsm-glossary)

## 21. Relational model

Способ представления данных через таблицы, ключи и связи между сущностями.

Углубиться:
- [PostgreSQL Data Definition](https://www.postgresql.org/docs/current/ddl.html)

## 22. Primary key

Поле или набор полей, который однозначно идентифицирует строку в таблице.

Углубиться:
- [PostgreSQL Constraints](https://www.postgresql.org/docs/16/ddl-constraints.html)

## 23. Foreign key

Ограничение, которое связывает одну таблицу с другой и поддерживает целостность ссылок.

Углубиться:
- [PostgreSQL Constraints](https://www.postgresql.org/docs/16/ddl-constraints.html)

## 24. PostgreSQL

Открытая реляционная СУБД, которая используется в программе для практики моделирования и реализации CMDB-модели.

Углубиться:
- [PostgreSQL docs](https://www.postgresql.org/docs/current/)

## 25. pgAdmin

Графический инструмент для работы с PostgreSQL: подключения, создания БД, таблиц и выполнения SQL.

Углубиться:
- [pgAdmin docs](https://www.pgadmin.org/docs/pgadmin4/latest/index.html)
