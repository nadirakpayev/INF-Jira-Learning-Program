# Knowledge Base

Эта папка нужна для теоретической подготовки кандидата перед выполнением практических заданий.

Она синхронизирована с тремя основными учебными контурами репозитория:
- [INF Learning Program/README.md](../INF%20Learning%20Program/README.md) — основной практический трек;
- [DataBase Fundamentals/README.md](../DataBase%20Fundamentals/README.md) — отдельный трек по РБД, SQL, PostgreSQL и pgAdmin;
- [Developers Theory Fundamentals/README.md](../Developers%20Theory%20Fundamentals/README.md) — отдельный трек по клиент-серверной архитектуре, HTTP и DOM.

Материалы разделены на два уровня:
- `базовый` — то, что нужно понять до начала первых блоков;
- `продвинутый` — то, что полезно изучать после освоения базы и по мере перехода к более сложным блокам.

## Как пользоваться

1. Начните с [01_Basic_Resources.md](01_Basic_Resources.md).
2. Затем пройдите [03_Basic_Glossary.md](03_Basic_Glossary.md).
3. После этого переходите к [02_Advanced_Resources.md](02_Advanced_Resources.md) и [04_Advanced_Glossary.md](04_Advanced_Glossary.md).

## Как это связано с треками

- Перед `Block 1` основного трека полезно пройти базовые материалы по `Jira Service Management Cloud`, полям, формам, очередям и SLA.
- Перед `Block 3` основного трека особенно важны базовые материалы по реляционным БД, SQL, `PostgreSQL` и `pgAdmin`.
- Перед `Block 4` основного трека особенно важны базовые материалы по клиенту, серверу, `HTTP`, `HTML` и `DOM`.
- Если в основном треке встречается незнакомый термин, сначала проверьте словари `Knowledge Base`, а затем при необходимости откройте профильный параллельный трек.

## Состав

- [01_Basic_Resources.md](01_Basic_Resources.md) — базовые внешние материалы
- [02_Advanced_Resources.md](02_Advanced_Resources.md) — продвинутые внешние материалы
- [03_Basic_Glossary.md](03_Basic_Glossary.md) — базовый словарь терминов
- [04_Advanced_Glossary.md](04_Advanced_Glossary.md) — продвинутый словарь терминов

## Примечание

- Ссылки подобраны и проверены `2026-04-11`.
- Для Jira Service Management часть возможностей зависит от плана Cloud. Это особенно важно для тем `major incident`, `problem management` и `change management`.
- В теоретической базе приоритет отдан официальным источникам Atlassian, PostgreSQL и pgAdmin.
