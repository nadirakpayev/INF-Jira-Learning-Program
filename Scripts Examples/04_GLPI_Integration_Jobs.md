# GLPI Integration Jobs

Источник:
- `Source Docs/Forte-J-ITSM25+Руководство+администратора_v1.0.docx`

В документе найдено описание интеграционных groovy-job’ов `GLPI -> Jira Assets`. Ниже сохранены все обнаруженные job’ы и вспомогательные классы.

## Основные job’ы

### `Job: актуализация групп ПО`

- Синхронизирует группы ПО из GLPI в Jira Assets
- Обновляет / создает:
  - `Прикладное ПО`
  - `Системное ПО`
- Использует:
  - `Config`
  - `GLPIAuth`
  - `GLPIService`
  - `MappingSoftware`

### `Job: актуализация ПК`

- Синхронизирует компьютеры типа `ПК`
- Фильтр:
  - `computertypes_id = 10`
- Особенности:
  - пакетная обработка;
  - ретраи при `AO/DB` ошибках;
  - ограничение числа итераций для тестов

### `Job: актуализация версионности`

- Синхронизирует:
  - версии ПО;
  - связи `ПК <-> Версионность`;
  - связи `Версионность <-> групповое ПО`
- Особенности:
  - many-to-many модель;
  - SQL к БД GLPI;
  - рекомендован `LIMIT N` для теста

### `Job: актуализация виртуальных машин (VM)`

- Синхронизирует `ComputerVirtualMachine` из GLPI в Assets
- Поиск совпадений:
  - сначала по `UUID`;
  - затем по комбинации `Имя + DNS`
- Есть:
  - retry;
  - timeouts;
  - логирование

### `Job: актуализация сетевого оборудования`

- Синхронизирует `NetworkEquipment`
- Поиск совпадений:
  - по `UUID`;
  - если не найдено — по серийному номеру
- Особенности:
  - чистка мусорных UUID;
  - агрегирование текстовых полей в описание;
  - `runWithAoRetry`

### `Job: актуализация Разрешенного ПО`

- Проходит по объектам `Версионность`
- Выставляет булевый атрибут `Разрешенное ПО`
- Источник истины:
  - объектный тип `Перечень разрешенного ПО`

## Mapping-классы

### `MappingComputer`

- Маппинг `GLPI -> Assets` для объекта `ПК`
- Создание и обновление объектов `ПК`

### `MappingSoftware`

- Маппинг записей ПО в `Системное ПО` и `Прикладное ПО`
- Есть метод `getSoftware(...)`

### `MappingVersion`

- Маппинг версий ПО в объект `Версионность`
- Создание и обновление версий

### `MappingVM`

- Маппинг виртуальных машин в объект `VM`
- Есть:
  - `createVMObject`
  - `virtualMachineMatching`
  - `getVM(...)`
  - `getDNSFromComment`
  - `getIPFromComment`

### `MappingNetwork`

- Маппинг сетевого оборудования в объект `Сетевое оборудование`
- Есть:
  - `createNetworkEquipmentObject`
  - `networkEquipmentMatching`
  - `getNetworkEquipment(...)`

## Вспомогательные классы

### `GLPIAuth`

- Получает `sessionToken` у GLPI
- Работает с:
  - `tokenUrl`
  - `appToken`
  - `userToken`

### `GLPIService`

- Вспомогательная логика:
  - `sanitize`
  - `escapeAqlString`
  - `getTotalCount(...)`

### `Mailer`

- Отправляет файлы логов по email после выполнения job’ов

### `Config`

- Централизованная конфигурация доступа к GLPI
- Хранит:
  - `tokenUrl`
  - `appToken`
  - `userToken`

## Операционные особенности, найденные в описании

- job’ы пишут логи в `$JIRA_HOME/log/glpi`;
- используются именованные логгеры `com.glpi.*`;
- есть retry и `Thread.sleep`;
- выполнение предполагается в background job-контексте;
- часть логики завязана на SQL к GLPI datasource;
- для тестов рекомендуется ограничивать объем обработки.
