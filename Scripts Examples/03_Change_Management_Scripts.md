# Change Management Scripts

Источник:
- `Source Docs/Forte-J-ITSM25+Руководство+администратора_v1.0.docx`

## `Автоматическая подстановка группы Change manager`

- Тип: field-setting script
- Назначение: заполнить поле `Группа Change Manager` по `Объект изменения`
- Логика:
  - извлечь `KE-...`;
  - найти систему в Assets;
  - получить связанную группу;
  - записать ее в Jira-поле

## `Заполнение поля Автостарт по плановой дате`

- Тип: derived field script
- Назначение: выставить `Да/Нет` по атрибуту шаблона в Assets

## `Заполнение Тип запроса для портала`

- Тип: fixed-value script
- Назначение: безусловно установить `Тип запроса клиента = Запрос на изменение`

## `Заполнение фактической даты`

- Тип: generic date setter
- Назначение: записать текущую дату или дату-время в указанное поле
- Особенность:
  - работает по `TARGET_CF_ID`;
  - поддерживает только `Date` и `DateTime`;
  - сохраняет без диспатча событий

## `Рассылка для согласующих`

- Тип: notification script
- Назначение: отправить письма согласующим RFC
- Источник получателей: поле `Согласующие RFC` + карточки сотрудников в Assets

## `Рассылка для участников группы Change manager`

- Тип: notification script
- Назначение: уведомить группу Change Manager о возврате изменения на доработку

## `Рассылка для участников группы исполнителей`

- Тип: notification script
- Назначение: уведомить группу исполнителей о завершении внедрения и начале тестирования

## `Создание связанной PIR задачи`

- Тип: issue creation helper
- Назначение: сформировать тему PIR и назначить исполнителя по полю `Исполнитель отклоненной подзадачи`

## `Создание связанной PIR-задачи_не успешно`

- Тип: issue creation helper
- Назначение: создать PIR с темой на основе шаблона и системы и проставить фиксированного `AM manager`

## `Создание связанной задачи PIR_условие`

- Тип: boolean condition
- Назначение: вернуть `true`, если плановая дата завершения просрочена

## `Создание события недоступности из RFC`

- Тип: issue creation script
- Назначение: создать связанную заявку `Событие недоступности` из RFC
- Делает:
  - читает систему и модуль;
  - проверяет влияние на доступность;
  - переносит плановые даты;
  - переносит Change Manager;
  - создает задачу в проекте `FORTE`

## `Увеличение счетчика откатов на +1`

- Тип: numeric post-function
- Назначение: инкрементировать `customfield_12329`
- Особенность:
  - рассчитан на Jira Data Center 10.x + ScriptRunner 9.x;
  - обновляет без диспатча событий

## Общие наблюдения

- Логика глубоко завязана на Assets, Data Center и ScriptRunner.
- Есть несколько жестко зашитых значений и строковых парсеров.
- Для Cloud эти сценарии потребуют переработки в automation + native fields + external mapping.
