# 02. SQL Basics

## Зачем нужен этот модуль

Этот модуль нужен, чтобы участник перестал воспринимать SQL как набор магических команд и начал видеть в нем обычный язык работы с данными.

## Что нужно понять

### 1. SQL как язык работы с данными

SQL используется для:
- создания структуры данных;
- вставки данных;
- чтения данных;
- изменения данных;
- удаления данных.

На практике для джуна особенно важны:
- `CREATE TABLE`
- `INSERT`
- `SELECT`
- `UPDATE`
- `DELETE`

### 2. SELECT

`SELECT` — базовая команда для чтения данных.

Нужно понимать:
- откуда берутся данные;
- какие столбцы выбираются;
- по каким условиям строки попадают в результат;
- как сортируется итог.

### 3. WHERE

`WHERE` отбирает только нужные строки.

Без этого знания невозможно:
- фильтровать данные;
- искать записи по условию;
- проверять корректность тестового датасета.

### 4. ORDER BY

`ORDER BY` задает порядок вывода.

Это особенно полезно:
- при проверке seed-данных;
- при чтении результатов запросов;
- при отладке модели.

### 5. JOIN

`JOIN` связывает строки из нескольких таблиц.

Это один из ключевых навыков для CMDB-модели, потому что:
- данные почти никогда не живут в одной таблице;
- реальные сущности связаны между собой;
- без `JOIN` нельзя увидеть полную картину.

### 6. Aggregate functions

Агрегатные функции помогают считать:
- количество;
- сумму;
- среднее;
- минимумы и максимумы.

Они нужны для:
- быстрой проверки данных;
- элементарной аналитики;
- построения validation queries.

### 7. UPDATE и DELETE

Нужно понимать не только чтение, но и изменение данных.

Особенно важно:
- `UPDATE` меняет существующие строки;
- `DELETE` удаляет строки;
- ошибка в условии может затронуть слишком много данных.

## Что нужно уметь после модуля

После этого модуля участник должен уметь:
- прочитать простой `SELECT`;
- написать базовый фильтр через `WHERE`;
- соединить две таблицы через `JOIN`;
- сделать простой `INSERT`;
- объяснить разницу между чтением и изменением данных.
