# 05. Glossary

## 1. Database

Организованное хранилище данных.

## 2. DBMS

Программная система, которая управляет базой данных.

## 3. PostgreSQL

Конкретная реляционная СУБД, используемая в программе как основная учебная платформа по БД.

## 4. pgAdmin

Графический инструмент для администрирования и работы с `PostgreSQL`.

## 5. Table

Структура хранения данных по строкам и столбцам.

## 6. Row

Одна запись в таблице.

## 7. Column

Один атрибут или признак, описанный в таблице.

## 8. Data type

Тип значения столбца: текст, число, дата и так далее.

## 9. Primary key

Поле или набор полей, которые однозначно определяют строку.

## 10. Foreign key

Поле или набор полей, которые ссылаются на строку в другой таблице.

## 11. Constraint

Правило, ограничивающее допустимые данные.

## 12. NULL

Специальное значение, означающее отсутствие известного значения.

## 13. Query

Запрос к базе данных.

## 14. SELECT

SQL-команда для чтения данных.

## 15. INSERT

SQL-команда для добавления данных.

## 16. UPDATE

SQL-команда для изменения данных.

## 17. DELETE

SQL-команда для удаления данных.

## 18. JOIN

Способ объединить строки из нескольких таблиц в одном запросе.

## 19. Entity

Тип объекта предметной области, который обычно моделируется отдельной таблицей.

## 20. Attribute

Свойство сущности.

## 21. Relationship

Логическая связь между сущностями.

## 22. Normalization

Подход к построению схемы, который уменьшает дублирование и делает данные более согласованными.

## 23. Transaction

Группа операций, которая выполняется как единое целое.

## 24. Commit

Подтверждение транзакции и сохранение результата.

## 25. Rollback

Отмена транзакции и возврат к предыдущему состоянию.

## 26. Index

Структура, ускоряющая поиск данных по определенным полям.

## 27. Seed data

Начальный набор данных для демонстрации и тестирования.

## 28. Validation query

Проверочный SQL-запрос, который подтверждает корректность схемы или данных.
